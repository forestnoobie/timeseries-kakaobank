## 2-1) 탐색적 데이터 분석(EDA)

EDA는 **막연한 탐색이 아닌 가설 검증** 형식으로 수행했습니다. `outputs/analytics/notebook.ipynb` (44 cells, Section 1~5) 참조. 산출물 요약은 `outputs/analytics/hypothesis_log.md`.

### 사전 등록 가설 9개와 검증 결과

| ID | 가설 | 검증 방법 | 핵심 수치 | 결론 |
|---|---|---|---|---|
| H1 | `holiday=1`은 value를 낮춘다 | Welch's t-test | t=60.4, p≈0, ratio=0.476 | **채택 ✅** |
| H2 | `event=1`은 value를 높인다 | Welch's t-test | t=-17.3, p=1.9e-37, lift=+42% | **채택 ✅** |
| H3 | 요일 효과 존재 | One-way ANOVA + Tukey HSD | F-test p<0.001, 다수 페어 유의 | **채택 ✅** |
| H4 | 연도별 추세(상승/하락) 존재 | Mann-Kendall (월별 평균) | τ=+0.884, p=5.5e-20 | **채택 ✅** |
| H5 | 강한 주간 계절성 (lag=7) | ACF/PACF | autocorr(lag=7)=+0.735 | **채택 ✅** |
| H6 | value 분포는 우편향 | Skewness, Shapiro-Wilk | skew=+0.69, Shapiro p≈0 | **부분 기각 ⚠️** (skew>1 미달) |
| H7 | 이상치 특정 일자 집중 | 3σ / IQR + 일자 패턴 분석 | 3σ outliers 식별, 월별 편향 확인 | **채택 ✅** |
| H8 | `holiday × dayofweek` 상호작용 | 2-way ANOVA 상호작용항 | p < 0.05 | **채택 ✅** |
| H9 | 결측 11건 비-MCAR | 결측 행 vs 전체 패턴 비교 | holiday rate 27% vs 32% (약한 차이) | **MCAR 가정 위배 증거 부족** |

### 핵심 인사이트 Top 5

1. **(H1)** holiday=1일 때 평균 value는 평일의 약 **47.6%** 수준으로 떨어진다 (Welch's t-test p≈0). 휴일이 가장 강한 단일 신호.
2. **(H4)** 월별 평균이 매우 강하게 **상승하는 추세** (Mann-Kendall τ=+0.884). 단순 계절 모델은 추세 흡수에 실패하므로 시간 인덱스 (`year`, `t_index`) 피처가 필수.
3. **(H5)** lag-7 자기상관 0.735로 매우 강한 주간 계절성. → `value_lag_{1,7,14,28}`, `rolling_mean_{7,28}` 피처로 반영.
4. **(H6 부분 기각)** skew=+0.69 (사전 기준 1.0 미달). 분포는 우편향이지만 강하지 않아 **log1p 변환은 옵션 토글로만** 유지 (default OFF).
5. **데이터 무결성**: `event` 컬럼에 스펙 외 값 (`event=2`) 1건 발견. 전처리에서 binary로 정규화 후 학습.

### EDA → 모델링 의사결정 로그

| EDA 결과 | 모델링 결정 | 코드 위치 |
|---|---|---|

## 2-2) 타겟 변수 예측 모형

### 평가 지표 선정과 근거

| 지표 | 채택 사유 | 비고 |
|---|---|---|
| **RMSE** | 메인 최적화 지표. 우편향+이상치 존재 → 큰 오차에 민감해야 비즈니스 리스크 감지 가능 | HPO 목적함수 |
| MAE | 이상치 영향 작음, 실무 해석 용이 | 보고용 |
| MAPE | 상대 오차 (value > 0) | 보고용 |
| sMAPE | MAPE 비대칭 보정 | 보고용 |
| R² | 분산 설명력 | 보고용 |

**메인 = RMSE 선정 근거**:
1. value 분포가 우편향+이상치 → 큰 오차에 페널티
2. M5 등 시계열 대회 표준
3. HPO surrogate 학습에 미분 가능한 형태 유리

**모든 모델은 `vs Naive seasonal y(t-7)` 개선율과 함께 보고** — 단일 지표 값보다 베이스라인 대비 개선이 중요.

### 모델 후보 선정

| 모델 | 역할 | 선정 근거 |
|---|---|---|
| Naive seasonal `y(t-7)` | 베이스라인 | H5 강한 주간 계절성 활용 |
| Group mean (요일×holiday) | 베이스라인 | 가장 단순한 휴리스틱. 어떤 모델이든 못 이기면 안 됨 |
| Ridge | 선형 베이스라인 | 비선형성 필요성 검증 (M1 가설) |
| **LightGBM** | **메인** | 시계열 회귀 표준 (M5 우승 레시피), 비선형/상호작용 자동 포착 |
| XGBoost | 보조 비교 | LightGBM과 다른 분할 전략 |

**검증**: `TimeSeriesSplit(n_splits=5)` (CV) + 시간순 train/val/test split (사전 등록·고정).
- train: 2018-01-01 ~ 2021-09-30 (45개월, 1,341일)
- val: 2021-10-01 ~ 2021-12-31 (3개월, 92일) — early stopping 용
- test: 2022-01-01 ~ 2022-03-31 (3개월, 90일) — 최종 평가용

### HPO — TuRBO (메인) + Optuna (비교군)

LightGBM의 9개 핵심 하이퍼파라미터에 대해 두 옵티마이저로 **공정 비교** 실행 (동일 9D 공간, 동일 `cv_objective`, 동일 예산 100 trials, seed=42 고정).

| 옵티마이저 | 알고리즘 | 라이선스 | 차별 이유 |
|---|---|---|---|
| **TuRBO** (메인) | Trust Region BO + GP (NeurIPS 2019, Eriksson+) | Uber Non-Commercial | 연구 알고리즘 이식 — 차별 포인트 |
| Optuna (비교) | TPE Sampler + MedianPruner | MIT | 산업 표준 |

**탐색 공간** (`src/tuning/param_spec.py:LGBM_SPACE`):

| # | 이름 | 종류 | 범위 |
|---|---|---|---|
| 1 | `learning_rate` | log_float | 1e-3 ~ 0.3 |
| 2 | `num_leaves` | int | 8 ~ 255 |
| 3 | `max_depth` | int | 3 ~ 12 |
| 4 | `min_child_samples` | int | 5 ~ 100 |
| 5 | `feature_fraction` | float | 0.5 ~ 1.0 |
| 6 | `bagging_fraction` | float | 0.5 ~ 1.0 |
| 7 | `bagging_freq` | int | 0 ~ 7 |
| 8 | `lambda_l1` | log_float | 1e-4 ~ 10.0 |
| 9 | `lambda_l2` | log_float | 1e-4 ~ 10.0 |

### 통합 성능 표 (test set, 2022-01 ~ 2022-03)

| Model | HPO | RMSE | MAE | MAPE | sMAPE | R² | vs Naive |
|---|---|---:|---:|---:|---:|---:|---:|
| Naive seasonal `y(t-7)` | — | 803.5 | 431.0 | 12.85% | 11.85% | 0.445 | 0.0% |
| Group mean (dow×holiday) | — | 821.0 | 782.5 | 21.52% | 24.30% | 0.421 | −2.18% |
| Ridge | default | 267.2 | 227.9 | 7.67% | 7.24% | 0.939 | +66.74% |
| LightGBM | default | 223.3 | 179.8 | 5.15% | 5.06% | 0.957 | +72.21% |
| XGBoost | default | 294.2 | 253.5 | 7.97% | 7.71% | 0.926 | +63.38% |
| LightGBM | Optuna 100t | 246.3 | 182.2 | 4.64% | 4.65% | 0.948 | +69.34% |
| **LightGBM** | **TuRBO 100t** | **204.3** | **146.7** | **3.79%** | **3.80%** | **0.964** | **+74.60%** |

> **최종 선정 모델**: LightGBM + TuRBO HPO. test RMSE **204.3**, R² **0.964**, Naive seasonal 대비 **+74.6% 개선**, default LightGBM 대비 추가 **+8.5% 개선**.

전체 metrics 파일: `outputs/predictions/metrics.csv` / val 평가: `metrics_val.csv`.

### TuRBO vs Optuna 정직 비교 (M4 가설 검증)

`outputs/hpo/comparison/report.md` 전체 리포트 + `convergence.png` 수렴 곡선.

| Optimizer | best CV-RMSE | walltime | test RMSE | test R² |
|---|---:|---:|---:|---:|
| **TuRBO** 100 evals | **456.81** | 33s | **204.25** | **0.964** |
| Optuna (TPE) 100 trials | 471.74 | 254s | 246.34 | 0.948 |

**M4 가설 부분 채택 ⚠️**: CV-RMSE 차이 3.27% (사전 정의 5% 미만 → 부분 채택), test에서는 TuRBO가 17% 우세.

> **본 9차원 LightGBM HPO에서 TuRBO가 우세**했지만, 이를 "TuRBO 절대 우세"로 일반화하지 말 것. TuRBO의 본 강점은 20~200차원 고차원 BlackBox 최적화 (NeurIPS 2019 원논문).
> 본 데이터에서 TuRBO 우세 요인: (1) ordered int는 사실상 quantized float, (2) log-scale 인코딩이 GP-friendly, (3) trust-region이 작은 budget에서 효율적, (4) 작은 데이터에서 optimizer overhead가 지배적.

## 실행 방법

### Docker (권장)

```bash
# 빌드 (~5분, TuRBO git 설치 포함)
docker build -t kakaobank2 .

# EDA 노트북만 실행
docker run --rm -v "$(pwd)/outputs:/app/outputs" kakaobank2 bash scripts/run_eda.sh

# 학습 + 예측 (HPO 제외 빠른 경로, ~30초)
docker run --rm -v "$(pwd)/outputs:/app/outputs" -v "$(pwd)/data:/app/data" kakaobank2 \
    python run.py all_no_hpo

# 전체 (EDA → preprocess → train → predict → HPO Optuna → HPO TuRBO → compare)
docker run --rm -v "$(pwd)/outputs:/app/outputs" -v "$(pwd)/data:/app/data" kakaobank2 \
    python run.py all
```

### 로컬 (Python 3.11 권장)

```bash
pip install -r requirements.txt
pip install "git+https://github.com/uber-research/TuRBO.git@master"
```

## 한계 및 향후 개선

- **외생 변수 부족**: holiday/event 외 마케팅·날씨·외부 이벤트 데이터 결합 시 잔차 자기상관 추가 감소 여지 (`outputs/predictions/residual_summary.csv`의 Ljung-Box).
- **코로나 효과**: 2020-03 이후 분포 이동 (강한 추세 H4의 일부일 수 있음). change-point 분석 미실시.
- **단일 시계열**: 다채널이라면 hierarchical/global 모델(N-BEATS, TFT 등) 검토 가치.
- **예측 구간 미제공**: 분위수 회귀(LightGBM quantile) 또는 부트스트랩으로 80% 예측 구간 추정 가능 — 현재 미구현.
- **Prophet 미포함**: 정성 분해 베이스라인. 패키지 무게 대비 가치 판단으로 제외.
- **HPO 단일 모델**: TuRBO/Optuna 모두 LightGBM에만 적용. XGBoost·Ridge에도 적용하면 추가 비교 가능하나, default 비교에서 LightGBM 우세 명확하여 우선순위 낮음.
